# Changelog

Все заметные изменения расширения **Claude Notify**.

Формат основан на [Keep a Changelog](https://keepachangelog.com/ru/1.1.0/),
проект придерживается [семантического версионирования](https://semver.org/lang/ru/).

## [2.0.0] - 2026-07-23

### Изменено (ломающее)
- Расширение переименовано `claude-ntfy-toggle` → **Claude Notify** (`claude-notify`).
- Ключи настроек переименованы `claudeNtfy.*` → **`claudeNotify.*`**. После
  обновления проверьте настройки: если раньше были заданы `claudeNtfy.topic`
  и др., перенесите их значения в `claudeNotify.*`.
- ID команд переименованы `claudeNtfy.*` → `claudeNotify.*`.

Файл-триггер (`~/.claude/.ntfy-trigger`) и сам хук Claude Code **не изменились**
— их править не нужно.

## [1.2.0] - 2026-07-23

### Добавлено
- Настройка `token` — Bearer-токен для защищённых топиков на собственном
  сервере ntfy.
- Настройка `priority` — приоритет пуша (`min`…`max`).
- Настройка `tags` — теги/эмодзи ntfy через запятую.
- Настройка `click` — URL, открывающийся по тапу на уведомлении.
- Таймаут HTTP-запроса (8 с), чтобы зависшее соединение не держало сокет.
- Резервный опрос файла-триггера — на случай, если `fs.watch` пропустит
  событие или каталог `~/.claude` появится уже после запуска.

### Изменено
- Команда «отправить тестовое уведомление» теперь ждёт ответ сервера и
  сообщает **реальный** результат: успех или понятную причину ошибки
  (неверный топик/токен, таймаут, нет сети).

## [1.1.0]

- Первый опубликованный вариант: кнопка-переключатель в статус-баре и пуш на
  телефон через ntfy при завершении запроса Claude Code.
